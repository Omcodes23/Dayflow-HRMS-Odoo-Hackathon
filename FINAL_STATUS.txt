## ðŸ“Š DATABASE TABLES

users table:
  - id (UUID)
  - email (string, unique)
  - name (string)
  - role (employee or hr)
  - company_id (optional)
  - created_at (timestamp)

companies table:
  - id (UUID)
  - name (string)
  - email (string)
  - industry (string)
  - employees_count (number)
  - owner_id (reference to users)
  - created_at (timestamp)

attendance table:
  - id (UUID)
  - user_id (reference to users)
  - date (date)
  - check_in (timestamp, optional)
  - check_out (timestamp, optional)
  - status (string)
  - created_at (timestamp)

leave_requests table:
  - id (UUID)
  - user_id (reference to users)
  - type (vacation/sick/personal)
  - start_date (date)
  - end_date (date)
  - reason (string)
  - status (pending/approved/rejected)
  - created_at (timestamp)
